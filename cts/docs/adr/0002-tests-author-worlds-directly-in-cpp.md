# Tests author their ANARI world directly in C++; the generic indirection layer is deleted

Each conformance test builds its ANARI world with normal ANARI C++ calls plus
shared generation helpers (`PrimitiveGenerator`, `TextureGenerator`,
`ColorPalette`, extracted from the old `SceneGenerator::commit()`). A slim
test-harness base supplies the essential conformance concepts (required
features, thresholds, bounds, permutation/variant, ground-truth binding).

The generic, JSON-driven indirection in the old `SceneGenerator` —
`createAnariObject(type, subtype)`, `setGenericParameter`,
`setReferenceParameter(refType, refIdx)`, and JSON-pointer parameter addressing —
is removed. That layer existed only so Python/JSON could drive ANARI object
construction from outside the process; with tests authored in C++ it is pure
accidental complexity.

We chose this over keeping the generic engine and merely moving test definitions
into C++ data/builders (lower risk, but preserves the indirection that made the
suite confusing) and over a hybrid that reused the old `commit()` bit-for-bit
(preserves rendered pixels, but keeps the engine we want gone).

Consequences: rewriting world construction may change rendered pixels, so
existing ground-truth reference images must be regenerated after migration. This
is acceptable because references are generated by a reference device (helide),
not hand-curated, and are not committed to the repository.
